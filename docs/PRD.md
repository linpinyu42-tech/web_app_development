# 讀書筆記本系統 - 產品需求文件 (PRD)

## 1. 專案概述

### 背景與動機
現代人常有閱讀的習慣，但在閱讀後往往缺乏一個簡單、專屬的空間來記錄自己對書籍的想法與評價。現有的系統可能過於複雜或充斥社交功能。因此，我們希望能建立一個純粹的「讀書筆記本系統」，讓使用者能方便地記錄自己看過的書，留存每一本書的精華，並追蹤自己的閱讀軌跡。

### 目標用戶
有讀書習慣的人。

### 核心價值主張
提供一個輕量、聚焦的筆記空間，幫助愛書人記錄過往閱讀的書本資訊、心得、分類與評分，方便未來的回顧與管理。

---

## 2. 功能需求

以下列出系統的主要功能，並以使用者故事（User Story）的格式描述：

1. **記錄書本資訊（書名與日期）**
   - 作為 **有讀書習慣的人**，我希望 **能夠記錄我讀過的書名以及相關日期紀錄**，以便 **追蹤我的閱讀進度與歷史軌跡**。

2. **撰寫讀後心得**
   - 作為 **有讀書習慣的人**，我希望 **能夠為每本書寫下自己的心得筆記**，以便 **完整留存與回顧當時閱讀時的體悟和重點**。

3. **書籍評分功能**
   - 作為 **有讀書習慣的人**，我希望 **能夠對每本書給予客觀的評分（如 1~5 星）**，以便 **未來能快速辨識與篩選出我最喜歡的推薦好書**。

4. **書籍分類管理**
   - 作為 **有讀書習慣的人**，我希望 **能夠為書籍設定分類（例如：文學小說、商業理財、科技工具等）**，以便 **按照不同的閱讀領域來收納和篩選我的筆記**。

5. **筆記列表與總覽**
   - 作為 **有讀書習慣的人**，我希望 **能夠在一個列表頁面查看我所有記錄的書本資訊（包含書名、日期、分類、評分與心得摘要）**，以便 **一目了然地回顧我的閱讀總結**。

---

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python 的 Flask 框架開發。
- **前端模板**：使用 Jinja2 引擎加上純 CSS / HTML 進行頁面渲染，確保輕量化。
- **資料庫**：使用 SQLite 作為關聯式資料庫，方便本機部署與測試。

### 效能與安全考量
- 確保所有表單輸入都經過基本驗證（防止 SQL 注入與 XSS 攻擊）。
- 列表頁面應維持順暢的操作體驗。

## 4. MVP 範圍 (Minimum Viable Product)

| 層級 | 功能描述 |
| --- | --- |
| **Must Have (必要)** | 1. 新增筆記（包含書名、心得、評分、分類、日期）<br>2. 筆記列表總覽頁面 |
| **Should Have (應該有)** | 1. 可點入單筆記錄查看完整心得<br>2. 編輯與刪除既有筆記的功能 |
| **Nice to Have (加分項)** | 1. 依照書籍分類進行資料篩選功能<br>2. 可視化的星等評分 UI |
